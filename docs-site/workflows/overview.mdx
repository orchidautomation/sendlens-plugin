# Workflows overview

Use `/sendlens-setup` first, then choose the specialist workflow that matches the decision you need: Analyst for broad diagnosis, Campaign Strategist for scale/pause/rewrite decisions, Copywriter for message analysis, Launch Operator for QA, and account briefs for client communication.

## Example prompt {#example-prompt}

```text
Use SendLens to answer this with provider scope, cache scope, evidence class, and unsupported surfaces: what should I do next?
```

## Evidence expectations {#evidence-expectations}

Link recommendations back to [the evidence model](/concepts/evidence-model). If evidence is sampled, inferred, reconstructed, or unsupported, say what would be needed to increase confidence.

## Read-only handoff {#read-only-handoff}

SendLens can recommend the next action, but provider changes must happen in a provider UI or approved native provider system of action.
