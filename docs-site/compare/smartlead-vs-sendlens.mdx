# Smartlead vs SendLens

Smartlead is the provider system of record and action. SendLens uses Smartlead read-only access to support local analysis and cross-provider normalization.

## Use Smartlead for {#use-smartlead-for}

- Campaign, lead, mailbox, sending, and account operations.
- Provider-native Smart Delivery configuration and support workflows.

## Use SendLens for {#use-sendlens-for}

- Read-only campaign, lead, reply, and sender analysis.
- Evidence-aware reporting inside an AI host.
- Comparison with Instantly workspaces where normalized fields are available.

## Smart Delivery caveat {#smart-delivery-caveat}

Smart Delivery and placement-adjacent surfaces can be support-gated. Unsupported or empty evidence from SendLens does not prove placement health.
