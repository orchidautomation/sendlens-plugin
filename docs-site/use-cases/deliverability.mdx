# Deliverability and sender risk

Look at bounce risk, stopped sending, sender availability, and support-gated placement surfaces. Empty unsupported evidence is not proof of health.

## Prompt pattern {#prompt-pattern}

```text
Analyze this use case in SendLens. Include data scope, provider scope, evidence classes, and any unsupported surfaces.
```

## Good output {#good-output}

A good answer separates exact counts from sampled examples, states uncertainty, and recommends a next human or provider-native action without claiming SendLens performed that action.
