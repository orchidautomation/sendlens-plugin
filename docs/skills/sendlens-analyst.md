# `sendlens-analyst`

`sendlens-analyst` owns diagnosis across performance, deliverability, replies, ICP, and copy. It also orchestrates the focused strategy, copywriter, and launch skills when a broad request spans the full chain.

## Use When

- The user asks what is working, failing, changing, or worth scaling.
- Deliverability, replies, ICP, copy, sequence, runway, or sender capacity need diagnosis.
- The user asks a broad question such as what is working and what to run, write, or launch next.

## Workflow

1. Start broad with workspace-level evidence when the question is broad.
2. Diagnose constraints in order: readiness, volume, senders/deliverability, reply quality, ICP/list quality, and copy mechanics.
3. Narrow to one campaign before deep reply, ICP, copy, or strategy work.
4. Validate reply quality and intended copy before promoting a winner.
5. Return validated findings, the primary constraint, the eligible opportunity, and material unknowns.
6. For broad requests, continue automatically through strategist → copywriter → launch operator.

## Boundaries

- Provider operations remain read-only.
- Exact, sampled, reconstructed, fetched, inferred, and unsupported evidence stay distinct.
- Focused campaign-blueprint requests use `sendlens-campaign-strategist`.
- Focused drafting and rewriting requests use `sendlens-copywriter`.
- Focused launch, scale/stop, measurement, or learning-handoff requests use `sendlens-launch-operator`.
- `sendlens-setup` owns installation, credentials, runtime, cache, and host-bundle troubleshooting.
