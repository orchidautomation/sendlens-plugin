# SendLens Decision Records

Use short decision records for behavior-changing SendLens work.

Create a record when a change affects:

- evidence classes or exactness language
- MCP response contracts
- startup/routing behavior
- sampling strategy
- reply hydration semantics
- client-safe output policy
- SendLens-vs-Pluxx ownership boundaries

Keep records short. A good record explains context, decision, consequences, and validation.

Use [TEMPLATE.md](./TEMPLATE.md) for new records.
